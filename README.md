# Synera — Auto Chess Arena

基于 **Qt6 + C++17 + MinGW** 的单机 PVE 自走棋对战游戏。玩家购买英雄、布阵，与敌方自动战斗，逐关推进。

## 快速开始

### 依赖

- Qt 6.x
- CMake 3.16+
- MinGW 64-bit (或 MSVC 2022)

### 构建

```bash
mkdir build && cd build
cmake .. -G "MinGW Makefiles"
mingw32-make
```

或使用 Visual Studio：

```bash
mkdir build && cd build
cmake .. -G "Visual Studio 17 2022"
cmake --build .
```

## 操作方式

| 操作 | 说明 |
|------|------|
| 鼠标点击招募槽 | 购买英雄（金币足够时英雄出现在回收槽，该槽变空） |
| 鼠标点击 Refresh 按钮 | 花费 15 金币刷新全部 5 个招募槽 |
| 鼠标点击 Pop+ 按钮 | 购买人口上限（招募区下方） |
| 鼠标拖拽 | 从回收槽拖拽英雄到棋盘（玩家半场 y≥4） |
| 拖拽到英雄信息面板 | 将英雄卖回（返还基础价格） |
| 拖拽到回收槽 | 将英雄存入回收槽备用 |
| 拖拽同整星同名叠加 | 升星合成（按整星匹配，非精确星等） |
| 开始战斗按钮 | 点击右侧按钮开始本关战斗 |
| **F5** | 保存游戏进度（准备阶段） |
| **F9** | 读取游戏进度（准备阶段） |
| **R** | 完全重置游戏 |

## 游戏机制

### 英雄类型

| 类型 | HP | 攻击/治疗 | 射程 | 移速/攻速(帧) | 技能 |
|------|----|:---------:|:----:|:-------------:|------|
| 战士 Warrior | 100×(整星+1) | ATK 20×(整星+1) | 1 | 60/120 | 对最近敌方造成 80×(1+整星×0.5) 伤害 |
| 法师 Mage | 50×(整星+1) | ATK 10×(整星+1) | 4 | 60/120 | 周围 5×5 敌方全体燃烧 4 回合 |
| 辅助 Support | 80×(整星+1) | 治疗 20×(整星+1) | 2 | 60/120 | 全场 HP 最低 2 个单位各回复 30×(1+整星×0.5) |
| 刺客 Assassin | 15×(整星+1) | ATK 50×(整星+1) | 1 | 60/80 | 锁定距离≤2 的最近敌方造成 80×(1+整星×0.5) 伤害 |
| Boss | 500 | ATK 20 | 1 | 120/120 | 技能1: 全体英雄各 10 伤害（3法力）; 技能2: 米字路径全体英雄各 30 伤害（5法力） |

> 移速/攻速值为帧间隔（已全局翻倍以优化伤害显示），值越小行动越快。刺客攻速最快（80帧）。
> **整星** = floor(半星等级 / 2)，即向下取整后的完整星数。

### 升星机制（半星制）

- 采用**半星制**：内部星等级 0-6 表示 0-3 颗完整星（每颗完整星 = 2 个半星）
- 两个**同名且同整星数**（`starLevel / 2` 向下取整相等）的英雄合成，结果为 `max(星等A, 星等B) + 1`（提高半颗星）
- **3 个相同英雄方可完成 1 整星升星**（两次合成）
- 拖拽到棋盘或回收槽中满足条件的英雄上即可升星
- **属性加成仅整星生效**：属性公式使用 `整星 + 1`（向下取整），半星无属性加成
- 星星显示：空星 / 左半实心（半星）/ 全实心（整星）
- 刺客升星后初始法力值满格

### 技能系统

- 每次普攻/治疗积攒 **1 点法力值**，满蓝自动释放技能（需有有效目标）
- **无有效目标时不释放技能**，法力值保留至下回合
- 攻击技能（战士/法师/刺客/Boss）仅对敌方释放
- 辅助技能仅对友方释放
- 若有效目标数不足，则对现有有效目标使用；无有效目标则保留技能点

#### 技能目标验证

| 职业 | 有效目标条件 |
|------|------------|
| 战士 | 任意敌方存活 |
| 法师 | 技能范围内有敌方（5×5 以自身为中心） |
| 辅助 | 有受伤友方（HP < maxHP） |
| 刺客 | 曼哈顿距离 ≤ 2 内有敌方（由独立刺客系统处理） |

### 战斗系统

- **帧级独立计时器**：每个单位有独立的 `m_moveTimer` 和 `m_attackTimer`
- 每帧计时器 +1，达到阈值时执行对应行动
- 移动计时器和攻击计时器完全独立，可同一帧同时触发
- 攻击范围内无目标时攻击计时器**每帧重置**，确保靠近后需等待完整攻速间隔

#### 战斗流程（每帧）

```
1. 清理过期特效
2. 收集棋盘存活单位
3. 燃烧结算（每 60 帧一次）
4. 刺客技能锁定处理（燃烧结算后立即执行）
5. 遍历所有单位：
   - 满蓝且非刺客 → 验证目标 → 释放技能或保留
   - 刺客 → 跳过主循环技能（由刺客系统处理）
   - 移动 & 普攻/治疗
6. 执行移动
7. 刷新伤害显示事件
8. 判定胜负
```

#### 燃烧 debuff

- 法师技能施加，每回合扣除 10 HP，持续 4 回合
- 独立于施法者生命周期
- 每 60 帧结算一次（燃烧间隔已翻倍）

### 寻路索敌

- **索敌规则**：欧几里得距离最近优先，距离相等时优先横向距离更近的
- **移动规则**：优先横向移动（x 方向），被阻挡则尝试纵向（y 方向）
- **辅助移动**：优先向右，其次纵向，最后向左
- 攻击性和辅助性角色使用相同索敌规则

### 刺客锁定机制

刺客技能**仅在燃烧结算帧处理**（每 60 帧一次），不通过主循环释放。

**锁定规则**：

1. 收集所有满蓝且范围内有敌方的刺客
2. 按棋盘顺序依次处理每位刺客：
   - 索敌（欧氏距离最近、横向破平，范围 ≤2）
   - **互杀检测**：若目标也是满蓝刺客 → 双方同归于尽，统计金币
   - **重复锁定**：若目标已被前序刺客击杀 → 自动重新索敌
   - 重索失败 → 本回合不释放技能，保留法力值
3. 对目标造成技能伤害并消耗法力值

### 伤害显示

- 每次攻击/治疗/技能产生 HP 变化后，在角色**右上角**显示浮动文本
- 格式：`+X`（绿色，治疗）或 `-X`（红色，伤害）
- 显示时长 = `max(最快攻速帧数 / 该单位本帧事件数 - 3, 0)` 帧
- 每帧结束后统一刷新，回合结束自动清空，不留残影

## 经济系统

### 金币获取

| 来源 | 金额 |
|------|------|
| 初始金币 | 1360 |
| 每关基础通关 | 100 |
| 击杀敌方 Warrior | 80 × (整星+1) |
| 击杀敌方 Mage | 60 × (整星+1) |
| 击杀敌方 Support | 30 × (整星+1) |
| 击杀敌方 Assassin | 30 × (整星+1) |
| 击杀 Boss | 200 × (整星+1) |
| 第 5 关前补助 | 800 |

- 玩家获胜：`m_gold += m_pendingGold + 100`
- 玩家失败（HP > 0）：`m_gold += m_pendingGold / 2 + 100`

### 金币利息

- 每关胜利后、添加战斗奖励前，**未使用金币变为 1.5 倍**
- 仅在**有下一关**时生效（即第 1-4 关胜利后）
- 失败或最终关不发放利息

### 英雄价格（波动制）

| 职业 | 基础价格 | 波动范围 |
|------|:----:|:----:|
| Warrior | 100 | 80 - 120 |
| Mage | 80 | 60 - 100 |
| Support | 50 | 30 - 70 |
| Assassin | 60 | 40 - 80 |

- 价格公式：`基础价格 + 5 × (rand()%5) × ((rand()%3) - 1)`
- 波动范围：基础价格 ±20（步长 5），最低 ≥ 1
- 价格在**每关准备阶段开始时**和**刷新招募区时**随机生成
- 招募区**实时显示当前波动价格**

## 人口上限

- 场上玩家方角色总数不能超过**人口上限**
- **初始人口上限**：5
- **升级费用**：`100 × (当前上限 - 4)` 金币
- **UI 显示**：棋盘右上角显示 `Pop: 当前人口/人口上限`
- **购买按钮**：左侧招募区下方「Pop+ $N」按钮
- 人口上限**跨关卡保留**（存档/读档支持）

## 招募系统

- 左侧面板分为**英雄信息区**（只读）和**招募区**（可交互）
- **英雄信息面板**（4 个）：显示每种英雄的基础属性（HP/ATK/Heal/射程/移速/攻速/价格），**拖拽英雄到对应面板可卖回**返还基础价格
- **招募区**（5 槽）：每槽随机显示英雄类型 + 波动价格，**点击直接购买**
- 购买后英雄**自动放入回收槽**空位（无空位则无法购买），该招募槽变空
- **刷新按钮**（Refresh $15）：花费 15 金币重随全部 5 个招募槽
- 招募区在每关准备阶段开始时自动刷新

## 关卡系统

| 关卡 | 敌方配置 | 总计 |
|:----:|---------|:----:|
| 1 | 每种类型各 1 个（0 整星，半星等=0） | 4 |
| 2 | 每种类型各 2 个（0 整星，半星等=0） | 8 |
| 3 | 每种类型各 3 个（0 整星，半星等=0） | 12 |
| 4 | 每种类型各 2 个（2 整星，半星等=4） | 8 |
| 5 | 每种类型各 1 个（3 整星，半星等=6）+ Boss | 5 |

- 仅一方**全部阵亡**时战斗结束
- 玩家 HP 初始 100，失败扣除 `剩余敌人数 × 20`
- HP ≤ 0 → **GAME OVER**
- 通过全部 5 关 → **VICTORY**

---

## 回收系统

- 回收槽：**2 行 × 8 列 = 16 槽**（与棋盘宽度一致）
- **双重用途**：① 战后回收英雄暂存 ② **从招募区购买英雄的暂存空间**
- 购买英雄后自动放入空回收槽（无空位则无法购买）
- 关卡胜利后自动回收场上存活英雄（治疗 20 HP，保留技能点）
- **回收优先级**：星数高优先 → 同星按 战士 > 法师 > 辅助 > 刺客
- 回收槽满时，剩余英雄丢弃
- 准备阶段可从回收槽拖拽英雄到棋盘或放回
- 回收槽常驻显示（空槽半透明边框，有单位则实心色块）
- 支持回收槽内升星合成

## 存档系统

- **F5** 保存（准备阶段），**F9** 读取（准备阶段）
- 存档文件：`savegame.json`
- 已有存档时**自动覆盖**（先删旧档再建新档）
- 存档内容：金币、关卡、HP、人口上限、招募区（类型/价格/空位）、棋盘所有英雄（位置/HP/法力/半星等/装备/燃烧）、回收槽所有英雄

## 代码架构

### 文件树

```text
PA/
├── CMakeLists.txt
├── README.md
├── main.cpp                   # 入口
├── synera.h / synera.cpp      # 主窗口、渲染、游戏循环、战斗、关卡/经济/回收
├── unit.h / unit.cpp          # Unit 基类、UnitType 枚举、速度/计时器、坐标/距离
├── hero.h / hero.cpp          # Hero + 四种职业英雄子类
├── enemy.h / enemy.cpp        # Enemy + 四种职业敌方子类 + Boss
├── weapon.h / weapon.cpp      # 装备类
└── board.h / board.cpp        # 8×8 棋盘数据
```

### 类层次

```
Unit (abstract)
├── m_type: UnitType {Warrior, Mage, Support, Assassin, Boss}
├── m_mana / m_mana2 / m_burning
├── m_moveSpeed / m_attackSpeed / m_moveTimer / m_attackTimer
├── getAttackRange() / getAttackDamage() / getHealAmount() / canHeal()
├── useSkill(board, allUnits) = 0    [纯虚：职业技能]
├── useSkill2(board, allUnits)       [虚：Boss进阶技能]
├── applyBurning() / tickBurning()
│
├── Hero ──── WarriorHero / MageHero / SupportHero / AssassinHero
└── Enemy ─── WarriorEnemy / MageEnemy / SupportEnemy / AssassinEnemy / BossEnemy

Board (8×8)  ◄── Unit*[8][8]

Synera (QMainWindow)
├── Board + vector<unique_ptr<Unit>> + vector<RecruitSlot> (招募区)
├── m_recycleSlots[16] (2行×8列回收槽 / 购买暂存)
├── m_hitEffects / m_slashEffects (战斗特效)
├── m_pendingDamageEvents (伤害累积显示)
├── 渲染: renderBoard / renderUnits / renderHeroInfo / renderRecruitment / renderRecycleSlots / renderUI
├── 战斗: processCombatFrame / processBurningTick / processAssassinSkills
├── 索敌: findNearestEnemyFor / findHealTarget / findNearestAlly
├── 结算: checkLevelEnd / endLevel / initLevel
└── 存档: saveGame / loadGame
```

### 关键枚举

```cpp

```
